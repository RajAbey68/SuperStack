# Buffer Idea Publisher

Use this to keep content ideas from disappearing inside chats.

## Workflow

1. Gather idea sources:
   - Recent research.
   - Codex memory or project notes.
   - Readwise saves.
   - YouTube transcript findings.
   - Draft scripts.
   - Current campaign priorities.
2. Select the strongest ideas.
3. Format each idea for Buffer or the target scheduler.
4. Add as ideas/drafts rather than publishing immediately unless explicitly asked.
5. Preserve source links or notes when available.

## Idea Format

For each draft idea include:

- Working title/hook.
- Platform fit: LinkedIn, X, YouTube Short, TikTok, newsletter, etc.
- One-line premise.
- Source/context.
- Suggested angle.
- Status: idea, draft, ready, or needs research.

## Rules

- Do not publish without explicit user approval.
- Avoid dumping weak ideas; curate.
- Keep Buffer clean by adding only ideas with a plausible next action.
- If API access is unavailable, produce an import-ready table or queue file.

## Automation Pattern

Useful recurring prompt:

```text
At the end of each day, review today's research and chats, choose five strong content ideas, and add them to Buffer as ideas with source context.
```
